# ⚖️ AI Legal Assistant with RAG System

A comprehensive, production-grade AI-powered legal assistant built with a native Node.js/TypeScript backend and a modern React/TypeScript frontend. The application features **Retrieval-Augmented Generation (RAG)** for highly accurate, citation-backed legal responses, dynamic legal search, real-time news aggregation, automated form filling, contract analysis, and multi-language support.

---

## 🚀 Features

### Core Functionality
- **RAG-Powered Legal Chat**: High-quality, grounded legal consulting with real-time semantic query generation, structured HTML responses, and key citations extracted dynamically from authoritative legal texts.
- **Contract Risk Assessment & Analysis**: Complete assessments identifying risk levels (Critical, High, Medium, Low), extracting key clauses, checking compliance, and offering actionable recommendations.
- **Multi-Document Comparison**: Side-by-side analysis using natural language processing to produce redline views and summary grids detailing core differences between agreements.
- **Intelligent Form Assistance**: Pre-filling and validating standard legal forms dynamically with clear, emoji-free guidance and legal citation references.
- **Regional News Aggregation**: Dynamic news feed powered by NewsAPI to query and load real-time legal developments across regions (India, US, UK, EU) and topics (Consumer protection, Corporate, Criminal, Property law), with progressive "Load More" pagination.
- **Semantic Legal Search**: Vector search-backed indexing to query legal contexts, with customizable categories and jurisdictions.

### Technical Capabilities
- **Robust Multi-Language System**: Full localization switching dynamically across English and Hindi (हिंदी)for all page text, guidelines, and components.
- **Fast Local Vector Store**: Efficient in-memory vector storage containing pre-embedded Indian statutory regulations, case laws, and guidelines.
- **Self-Healing LLM Backends**: Fail-safe mechanisms routing requests to Groq (Llama 3.3/3.1) with prompt fallback structures designed to ensure robust, valid legal information.
- **Privacy & Security Control**: User-controlled session data lifespan and toggleable local storage query anonymization settings.

---

## 📊 Integrated Legal Datasets

Our vector database contains embedded context from leading legal databases and primary regulatory acts:
1. **HuggingFace Indian Law Dataset** (`viber1/indian-law-dataset`): Primary constitutional provisions, court precedents, and statutory regulations.
2. **Contract NLI Dataset** (`kiddothe2b/contract-nli`): Contract agreements, clause annotations, and risk factors.
3. **JusticeHub Indian Judiciary Data** (`justicehub.in`): High Court and Supreme Court statistics and judicial structures.
4. **Primary Indian Acts**: Right to Information Act (2005), Consumer Protection Act (2019), Indian Penal Code (IPC), and Indian Contract Act (1872).

## 🛠️ Installation & Setup

### Prerequisites
- **Node.js** (v18 or higher recommended)
- **npm** or **yarn**
- **API Keys**: A Groq API key and a NewsAPI key.

---

### 1. Clone & Install Dependencies

Clone the repository and install the dependencies:

```bash
# Clone the repository
git clone <repository-url>
cd AI-Law-Bot

# Install root dependencies
npm install

# Install frontend dependencies
cd frontend
npm install
```

---

### 2. Configure Environment Variables

Create a `.env` file in the **root** folder:

```bash
# Copy template env
cp env.example .env
```

Open the `.env` file and set your keys (OpenAI is not required):

```env
# Server Configuration
PORT=3001

# AI API Key (Groq Llama 3.3/3.1)
GROQ_API_KEY=gsk_your_groq_api_key_here

# Real-time News API Key (Sign up at newsapi.org)
NEWS_API_KEY=your_news_api_key_here
```

---

### 3. Running the Application

To start the application, open two terminal sessions:

#### 1. Start the Backend Server (Runs from Root)
In the root directory, launch the backend:
```bash
npm start
```

#### 2. Start the React Frontend (Runs from frontend)
In a separate terminal, navigate to the frontend directory and launch the client:
```bash
cd frontend
npm run dev
```

#### Access Links
- **Frontend Client**: http://localhost:8080 (or the port generated by Vite)
- **Backend API**: http://localhost:3001
- **Health Check**: http://localhost:3001/health

## 🔧 API Endpoints Reference

### 🌐 System Endpoints
- `GET /health` - Check backend availability, active RAG pipeline, and API health status.
- `GET /api/metrics` - Real-time stats on documents parsed, sessions logged, and query counts.

### 🧠 AI & RAG Endpoints
- `POST /api/chat` - RAG Chatbot. Body: `{ message: string, language: string, sessionId?: string }`
- `POST /api/search` - Semantic legal search querying primary databases.
- `POST /api/documents` - PDF/txt document upload and RAG indexing.

### ⚖️ Advanced Tooling
- `POST /api/contracts/analyze` - Assessment of agreements for compliance and risk level metrics.
- `POST /api/forms/fill` - Logic-based legal form field validation.
- `POST /api/documents/compare` - Document differences analysis yielding structured summary redlines.
- `GET /api/news` - News API proxy. Parameters: `?region=India&topic=general&limit=6`

## 🔒 Security, Compliance, & Privacy

- **On-Demand Anonymization**: Allows users to enable query pre-processing which blocks storing PII local storage footprints.
- **Data Deletion Control**: User-defined slider policies on the UI (from 1 day to 365 days) allowing the system to automatically purge outdated chat logs.
- **Role-Based Protection**: All data requests and files uploaded utilize isolated session tracking tokens to enforce clean customer-level sandboxing.
